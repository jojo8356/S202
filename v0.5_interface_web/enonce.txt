  La mise en oeuvre de l'interface web nécessite d'implémenter un serveur Http , avec une sorte d'API
  Rest sommaire.

  Quel est le principe ?

  L'idée est est que la commande herve web va lancer un serveur http, qui tournera ensuite en arrière plan
  sur le port 2026.

  (Le numéro de port a été fixé arbitrairement, afin que tous les groupes utilisent le même port).

  On va ensuite pouvoir passer des paramètres en utilisant des requêtes GET.

  Exemple : http://127.0.0.1:2026/?recherche=recette+du+gaspacho fournit les résultats de la
  recherche "recette du gaspacho"

  Voici un exemple complet, en partant de la ligne de commande :

  J'exécute la commande suivante :

  herve web --recherche "recette du gaspacho"


  La méthode main de herve.Herve va donc être appelée, puis rediriger vers la méthode appropriée.

  Celle-ci va :

        démarrer le serveur si besoin (le plus simple est de regarder si une requête GET à
        http://127.0.0.1:2026/test renvoie bien "OK").
        Si ce n'est pas le cas, elle le démarre automatiquement.
        Envoyer une requête GET à http://127.0.0.1:2026/?recherche=recette+du+gaspacho
  Ceci va provoquer l'appel du HttpHandler du serveur (cf. https://blog.ronanlefichant.fr/2019/05/java-http-
  server.html  par exemple).

  Celui-ci va découper l'URI, récupérer le path de l'URI (en principe /) et surtout les paramètres de la requête
  GET contenus dans l'URI (ici, recherche=recette+du+gaspacho).

        On récupère ensuite les résultats de la recherche correspondante et on renvoie le code HTML et le
        code CSS appropriés.

  Remarque sur le lien fourni plus haut (https://blog.ronanlefichant.fr/2019/05/java-http-server.html ) :
  lorsque j'ai testé le code présenté sur le blog, il ne fonctionnait pas car la méthode sendResponseHeaders
  attend un nombre d'octets, et non un nombre de caractères. Il faut donc remplacer
  exchange.sendResponseHeaders(200, response.length()) par exchange.sendResponseHeaders(200,
  response.getBytes("UTF-8").length)


  (Rajoutez également une entête Http pour indiquer l'encodage au navigateur).
