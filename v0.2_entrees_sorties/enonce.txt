  Format des fichiers d'indexation
  À chaque page est associé un fichier d'index.

  Son format est très simple :

        La 1re ligne contient seulement l'URL de la page web correspondante.
        Chaque ligne suivante contient un mot du document, suivi de ":" et du nombre d'occurrences du mot.


  Ces fichiers sont au format UTF-8.



  Vous devez implémenter le constructeur manquant de la classe IndexPage, qui permet de générer le
  contenu d'indexation à partir d'un fichier d'index présent sur le disque (cf. diagramme de classe ).

  Comme les fichiers sont petits, vous pouvez pour l'instant vous contenter d'utiliser Files.readAllLines ()
  pour lire le contenu du fichier (vous verrez ulétrieurement en TD d'autres méthodes plus efficaces,
  notamment pour de gros fichiers).


     Remarque :
     Lire un fichier sur le disque est susceptible de générer une erreur d'entrée-sortie (par exemple, parce
     qu'il n'y a pas de fichier à l'emplacement demandé, ou parce que le programme n'a pas la permission
     d'y accéder).
     En Java, dès qu'une méthode est susceptible de générer une erreur (to raise an exception), elle doit
     soit gérer le cas d'erreur, soit indiquer dans sa signature la possibilité qu'une erreur soit générée.
     Pour l'instant, vous pouvez simplement ajouter throws IOException à la signature du constructeur
     en question (Eclipse vous proposera de le faire).

  Classe principale
  Vous devez maintenant implémenter les classes SearchResult et SearchEngine, telles que décrites dans
  le diagramme de classe présenté à l'étape précédente .

  Toutes les méthodes publiques doivent être testées dans SearchEngineTests.

  La classe SearchEngine est la classe principale qui a pour fonction de lancer une recherche, via sa
  méthode launch_request().

  En particulier, le constructeur de la classe SearchEngine doit lire tous les fichiers index du dossier
  indexation_directory, et générer un tableau (array) de pages indexées (IndexedPage), stockées dans
  l'attribut privé pages.

  (Remarque : lorsque vous implémenterez le constructeur de SearchEngine, java vous préviendra là encore
  qu'une IOException est susceptible d'être générée.)

  La méthode printResults() doit afficher les 15 résultats les plus pertinents d'une recherche - du plus
  pertinent au moins pertinent. Les résultats de pertinence nulle ne doivent pas apparaître, donc il peut
  éventuellement y avoir moins de 15 résultats affichés.

  Interface
  Rajouter dans la classe SearchEngine une méthode main, qui servira d'interface au programme.

        Si le programme SearchEngine est appelé avec des arguments, ces arguments sont interprétés
        comme une requête.
        Dans ce cas, printResults() est appelée pour afficher les résultats de cette requête et le programme
        s'arrête.
        Si le programme SearchEngine est appelé sans argument, on bascule en mode interactif.
        Une invite de commande doit permettre à l'utilisateur de saisir une requête.
        On récupère les mots rentrés par l'utilisateur.
        Si l'utilisateur a tapé exit, on quitte.
        Sinon, on interprète les mots tapés comme une requête et on affiche les résultats de la recherche,
        puis on recommence.



  Testez son bon fonctionnement dans le shell (par ex. Powershell sous Windows ou Bash sous Linux) :

        mode interactif : java -cp . search_engine.SearchEngine
        mode "one-shot" : java -cp . search_engine.SearchEngine flan cerise
